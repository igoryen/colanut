Settings How To Overview [01]

Welcome to the Connexions Admin Settings User Guide. The Settings options have been designed to assist in making Connexions more flexible and user friendly. The data and configurations have been organized to allow more visibility, organization, and usability. The configurations have also been segregated, so they can be set at two different levels: At the Site level, which would be beneficial in the instance that the entire site has one workflow or at a Location level, which would be beneficial if there are multiple channels of business on one site that have different workflows.

As you review the Settings, you will notice that some sections are duplications allowing the Admin user to access the same section by a different path. The design is to simplify the workflow allowing the Admin user to complete their task (i.e.: creating a user profile or changing a configuration) as quickly and seamlessly as possible.

Click the Online Help button for general help inquiries for System Admin users and to review frequently asked questions (FAQ). Lender and vendor users also have a version of Help to assist with their inquiries.

The link in Connexions takes you to the Settings site, which is configured based on the lender’s requirements. Only users with the Admin role will have access to this site. Within Settings, user profiles are created with their roles/permissions granted, panels are built, pricing is set, and guidelines are created.

The Settings page is divided into three separate sections: Selector, Sub-Selector, and Action panes.

Selector Pane [01-001]

The Selector pane at the far left of the screen is separated into sections; each section applies to a different type of user on the site.

  • CX Internal
  • Site (lender)
  • Appraiser
  • Appraisal Company
  • AMC (Appraisal Management Company)
  • Broker
  • Broker Company
  • External Review
  • Investors
  • Correspondent

Sub-Selector Pane [01-002]

The Sub-Selector pane (center of the screen) is driven by the task chosen from the Selector pane. For example, if Manage Site Users is selected from the Selector pane, then a list of users will display in the Sub-Selector pane. If Manage Locations is selected, then a list of locations will display in the Sub-Selector pane.

Action Pane [01-003]

The Action pane (far-right portion of the screen) is driven by the task chosen from the Selector pane and the object chosen from the Sub-Selector pane. If Manage Site Users is selected in the Selector pane and a specific user is selected in the Sub-Selector pane, then the Action pane will display the selected user’s information. If Manage Locations is chosen in the Selector pane and a specific location is selected in the Sub-Selector pane, then the Action pane will display location information. Additional sections may be expanded to manage the location panel, pricing, and UCDP/EAD.

Within any screen that does not have an implicit Save or Delete button, clicking outside of the area prior to closing the section or exiting Settings will automatically save the changes.

Manage Pricing [02-001]

Navigation: Selector = Manage Pricing / Sub-Selector = Desired Location / Action = Manage Pricing

Manage Pricing features a streamlined display with the following capabilities:

  • Setting the lender pricing at each location
  • Importing or exporting an entire price sheet
  • Add, delete or edit pricing of an individual product

You can manage the pricing of each product. When you expand the menu, the amounts are shown for each individual product, by state.

To edit a product’s amount, click the edit symbol you see when you hover over the amount that needs to be changed. Change the amount and click the check mark to save the change. You can cancel the entry by clicking the x.

Delete any entry by clicking the x you see when you hover over the column.

Other tasks you can do from the Manage Pricing page:

  • Add new pricing by clicking the sign. Then, from the screen that displays, select a product from the Product drop-down list, the State, and County. Enter an Amount and click Save.

  • Import new pricing by clicking the sign. Click the File button and search for and select the file you wish to import. Click Process. A message will display: “Please wait…we are preparing your file during the processing.”

  • Export the current pricing by clicking the sign. A message will display showing your progress. The exported pricing file will display at the bottom of the screen, when finished.

Manage Note Delivery Groups [02-004]

These settings allow the management of note delivery groups to customize communications. CX Internal Admin users can configure these options to customize by role who can view notes on the Appraisal Details page and who will receive email notifications. This decreases the number of unwanted notes, while maintaining access of the notes for those roles that need to see them. The CX Internal Admin can also edit and delete notes. These settings are read-only for non CX Internal Admin users.

The available Note Delivery Groups are as follows (there are standard roles assigned to each group, but additional roles can be added):

  • User Assigned to Order
  • AMC
  • Appraiser
  • Lender
  • AMC Platform
  • Appraisal Desk
  • Log
  • Log Internal
  • Escalation Team
  • Payment Users

Add New Roles to Group Delivery [02-004-001]

Navigation: Selector = Manage Note Delivery Groups / Sub-Selector = N/A / Action = Add New Roles to Group Delivery

  1. Click on the icon to open a form to select delivery options.

  2. Select one of the Group Delivery options. Note that each of the selection choices have detailed sections below where you can see the Role that is associated with the option and whether the roles can view notes and/or receive email notifications.

  3. Select the Role you would like added to the Group.

  4. Select Yes in the Can View Note field if you want the selected role to be able to view notes assigned to the order.

  5. Select Yes in the Get Notification field if you want the selected role to receive notes assigned to the order through an email notification.

  6. Click Save.

The role is added to the Delivery Group. From the grid for the Delivery Group, you can Delete or Edit your changes.

Vendor Assignment Type Logic [08-001-004]

Connexions Vendor Assignment Type Logic

The logic behind the Round Robin, Weighting, and Random assignment methods is described in the table below.

Round Robin

Round robin type assignment

Each vendor is lined up and a vendor is assigned consecutively.

  • The Assignment Engine pulls the vendor’s Geopoints and compiles them into a list of qualified vendors taking into consideration license, loan type (FHA), away, E&O config, etc., monthly maximum, assignment threshold, allowed property type, allowed loan types, allowed product types.
  • It checks which vendor was assigned previously using round robin and assigns the order to the next consecutive vendor on the list (automatic only, manually assignment does not count).
  • If an order is manually reassigned from a round robin vendor chosen by the Assignment Engine, the vendor will have to wait for the next round after exhausting the current round.
Weighting

Raffle type assignment

The higher the weighting, the higher the chance a vendor is picked for assignment.

  • The Assignment Engine pulls the vendor’s Geopoints and compiles them into a list of qualified vendors taking into consideration license, loan type (FHA), away, E&O config, etc., monthly maximum, assignment threshold, allowed property type, allowed loan types, allowed product types.
  • Similar to purchasing raffle tickets, a string of numbers is generated from the total weight (sum of all qualified vendor weightings) and assigned to each vendor.○ The higher the weighting, the more “tickets” the vendor gets, thus increasing the vendor’s probability.
  • The Assignment Engine chooses a number and the vendor with the matching number is assigned the order.
  • Manual reassignment does not affect the vendor assignment chances, only the vendor weighting.
Random

Lottery type assignment

Each vendor gets one number and the system randomly picks a number.

  • The Assignment Engine pulls the vendor’s Geopoints and compiles them into a list of qualified vendors taking into consideration license, loan type (FHA), away, E&O config, etc., monthly maximum, assignment threshold, allowed property type, allowed loan types, allowed product types.
  • Each vendor is assigned a number.
  • The Assignment Engine randomly chooses a number and assigns the order to the vendor with the matching number.
  • Manual reassignment does not affect the vendor assignment chances.

Weighted Feature Example [08-001-005]

If a vendor has a weight of 100 and another vendor has a weight of 50, the chances of the first vendor being selected is most likely. It is comparable to a lottery drawing and for every number above the second vendor (in this example, 50), that is the number of tickets the first vendor has in the pool. This means the vendor with this highest number of tickets is most likely to be drawn for the order assignment.

Order assignment takes into consideration the Connexions settings for in Manage Assignment Criteria (Appraiser/ Appraisal Company/AMC > Manage Assignment Criteria). The Monthly Maximum option determines the number of orders to be assigned in a month. The Assignment Threshold option sets the number of orders to be assigned and opened at the same time. Account work history such as late orders or early orders do not factor in to the assignment.
